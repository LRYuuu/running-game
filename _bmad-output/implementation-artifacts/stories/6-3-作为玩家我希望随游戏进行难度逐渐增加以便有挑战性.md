## Story

作为玩家，
我希望随游戏进行难度逐渐增加，
以便有挑战性。

## Acceptance Criteria

### 功能验收标准

1. **难度随距离递增**
   - Given 游戏进行中
   - When 玩家存活距离/时间增加
   - Then 障碍物密度逐渐增加

2. **难度曲线可控**
   - Given 开发者在 DifficultyConfig 中配置参数
   - When 游戏运行时
   - Then 难度按照配置的曲线增长

3. **障碍物生成概率动态调整**
   - Given 初始障碍概率为 30%
   - When 游戏进行到一定距离后
   - Then 障碍概率逐渐上升到最大值（如 60%）

4. **空隙生成概率动态调整**
   - Given 初始空隙概率为 10%
   - When 游戏进行到一定距离后
   - Then 空隙概率逐渐上升到最大值（如 20%）

5. **游戏速度动态增加（可选）**
   - Given 游戏有滚动速度机制
   - When 游戏进行时间增加
   - Then 游戏速度逐渐加快

6. **最小难度保护**
   - Given 游戏刚开始
   - When 距离为 0
   - Then 使用最低难度（前 3 个 Chunk 保持安全）

7. **最大难度限制**
   - Given 游戏进行很长时间
   - When 距离超过阈值
   - Then 难度不再无限增加，保持在最大值

### 技术验收标准

1. **难度计算独立模块**
   - Given 难度系统需要被多个模块使用
   - When 其他模块需要当前难度值
   - Then 通过单例或事件提供难度数据

2. **难度事件通知**
   - Given 难度等级变化时
   - When 难度值跨越阈值
   - Then 触发 OnDifficultyChanged 事件

3. **与现有配置系统集成**
   - Given 已有 GameConfig 和 TilemapMapConfig
   - When 难度系统影响生成参数
   - Then 通过事件或回调通知生成器

4. **难度可视化调试**
   - Given 开发模式下运行
   - When 游戏进行中
   - Then 可在 UI 或日志中看到当前难度值

5. **可测试性**
   - Given 难度计算逻辑
   - When 编写单元测试
   - Then 可以独立测试难度曲线函数

### 核心算法伪代码

```csharp
// 难度计算核心逻辑
public class DifficultyCalculator : MonoBehaviour
{
    [SerializeField] DifficultyConfig config;

    public float CurrentDifficulty { get; private set; } // 0.0 - 1.0
    public float CurrentObstacleChance { get; private set; }
    public float CurrentGapChance { get; private set; }

    void Update()
    {
        float distance = GetPlayerDistance();
        UpdateDifficulty(distance);
    }

    void UpdateDifficulty(float distance)
    {
        // 安全区域：前 3 个 Chunk 不增加难度
        if (distance < config.startDistance)
        {
            CurrentDifficulty = 0f;
            CurrentObstacleChance = config.baseObstacleChance;
            CurrentGapChance = config.baseGapChance;
            return;
        }

        // 计算难度进度 (0.0 - 1.0)
        float progress = Mathf.InverseLerp(
            config.startDistance,
            config.maxDistance,
            distance
        );

        // 应用曲线类型
        switch (config.curveType)
        {
            case CurveType.Linear:
                CurrentDifficulty = progress;
                break;
            case CurveType.Exponential:
                CurrentDifficulty = Mathf.Pow(progress, 2); // 先慢后快
                break;
            case CurveType.Logarithmic:
                CurrentDifficulty = Mathf.Log(progress + 1, 2); // 先快后慢
                break;
        }

        // 计算当前障碍/空隙概率
        CurrentObstacleChance = Mathf.Lerp(
            config.baseObstacleChance,
            config.maxObstacleChance,
            CurrentDifficulty
        );

        CurrentGapChance = Mathf.Lerp(
            config.baseGapChance,
            config.maxGapChance,
            CurrentDifficulty
        );

        // 触发事件
        OnDifficultyChanged?.Invoke(CurrentDifficulty);
    }
}
```

## Implementation Notes

### 实现建议

1. **难度曲线调优**
   - 建议从线性曲线开始实现，简单易调优
   - 预留指数/对数曲线接口，方便后续迭代
   - 使用 AnimationCurve 可以让策划在 Inspector 中自定义曲线

2. **障碍概率计算**
   ```csharp
   // 在生成器中使用时
   float currentChance = difficultyCalculator.CurrentObstacleChance;
   if (Random.value < currentChance)
   {
       SpawnObstacle();
   }
   ```

3. **空隙生成的额外限制**
   - 即使难度增加，也要保证最小间隔
   - 连续空隙之间至少需要 N 个安全 Chunk

4. **性能优化**
   - 难度计算不需要每帧执行，可以每秒更新一次
   - 使用事件驱动，仅在难度变化时通知

5. **调试支持**
   - 在开发模式下显示：`Difficulty: 0.75 (Obstacle: 45%, Gap: 15%)`
   - 使用 `Debug.DrawLine` 可视化难度变化

### 默认配置建议

```csharp
// DifficultyConfig 默认值
public class DifficultyConfig : ScriptableObject
{
    [Header("难度起点")]
    [Tooltip("开始增加难度的距离（米）")]
    public float startDistance = 50f;  // 前 50 米安全

    [Header("难度顶点")]
    [Tooltip("达到最大难度的距离（米）")]
    public float maxDistance = 500f;   // 500 米达到最大难度

    [Header("障碍概率")]
    [Tooltip("基础障碍概率")]
    [Range(0f, 1f)]
    public float baseObstacleChance = 0.3f;

    [Tooltip("最大障碍概率")]
    [Range(0f, 1f)]
    public float maxObstacleChance = 0.6f;

    [Header("空隙概率")]
    [Tooltip("基础空隙概率")]
    [Range(0f, 1f)]
    public float baseGapChance = 0.1f;

    [Tooltip("最大空隙概率")]
    [Range(0f, 1f)]
    public float maxGapChance = 0.25f;

    [Header("曲线类型")]
    [Tooltip("难度增长曲线类型")]
    public CurveType curveType = CurveType.Linear;
}

public enum CurveType
{
    Linear,      // 线性增长
    Exponential, // 先慢后快（适合 Roguelike）
    Logarithmic  // 先快后慢（适合休闲）
}
```
