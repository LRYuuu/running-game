## Story

作为系统，
我可以在死亡时重置当前分数，
以便公平计分。

---

## Acceptance Criteria

### 功能性验收标准

1. **死亡时分数重置**
   - Given 游戏处于 `Playing` 状态
   - When 玩家死亡，GameManager 切换到 `Dying` 状态
   - Then 当前分数立即重置为 0

2. **重生时分数保持为零**
   - Given 玩家已死亡
   - When 重生流程完成，游戏状态切换到 `Playing`
   - Then 分数从零开始重新累加

3. **最高分不受影响**
   - Given 玩家死亡
   - When 分数重置时
   - Then 历史最高分保持不变

### 技术性验收标准

1. **与 GameManager 集成**
   - Given 玩家死亡
   - When GameManager.OnPlayerDied() 被调用
   - Then GameManager 通知 ScoreManager 重置分数

2. **与 ScoreManager 集成**
   - Given ScoreManager 已初始化
   - When 游戏状态切换到 `Dying` 或 `Waiting`
   - Then 自动调用 `ResetScore()` 方法

3. **事件触发**
   - Given 分数被重置
   - When `ResetScore()` 被调用
   - Then 触发 `OnScoreChanged` 事件，参数为 (0, HighScore)

## Implementation Notes

### 方案 A: 通过 GameManager 状态机自动触发（推荐）

修改 `ScoreManager.OnGameStateChanged()` 方法：

```csharp
private void OnGameStateChanged(GameState oldState, GameState newState)
{
    switch (newState)
    {
        case GameState.Playing:
            _isAccumulating = true;
            _accumulationTime = 0f;
            break;
        case GameState.Dying:
            _isAccumulating = false;
            CheckHighScore();
            ResetScore();  // ←── 添加此行
            break;
        case GameState.Waiting:
            _isAccumulating = false;
            ResetScore();
            break;
    }
}
```

### 方案 B: 在 GameManager.StartRespawn() 中显式调用

修改 `GameManager.StartRespawn()` 协程：

```csharp
private IEnumerator StartRespawn()
{
    if (_enableDebugLog)
    {
        Debug.Log("[GameManager] 开始重生流程...");
    }

    yield return new WaitForSeconds(respawnDelay);

    ChangeState(GameState.Respawning);

    // 1. 清理地图和障碍物
    if (_mapGenerator != null)
    {
        _mapGenerator.Cleanup();
    }

    // 2. ★ 添加：重置分数 ★
    // 注意：ScoreManager 可能为空（如果 Story 4-2 尚未实现）
    // 可以通过调用 PlayerDeathController.Respawn() 来间接重置
    // 或者检查 ScoreManager.Instance 是否存在

    // 3. 重置玩家跳跃状态
    if (_playerJumpController != null)
    {
        _playerJumpController.ResetJumpState();
    }

    // 4. 重置玩家位置
    if (_playerDeathController != null)
    {
        _playerDeathController.Respawn();
    }

    // 5. 重新生成地图
    if (_mapGenerator != null)
    {
        _mapGenerator.Initialize();
    }

    ChangeState(GameState.Playing);
}
```

### 推荐实现

**使用方案 A**，因为：
1. 代码更简洁，符合事件驱动架构
2. 自动处理所有状态转换场景
3. 不需要修改 GameManager，减少耦合
