## Story

作为系统，
我可以播放碰撞音效，
以便明确死亡。

---

## Acceptance Criteria

### 功能验收标准

1. **碰撞时播放音效**
   - Given 游戏进行中
   - When 玩家与障碍物发生碰撞时
   - Then 播放碰撞音效

2. **音效音量控制**
   - Given 碰撞音效正在播放
   - When 调整音效音量设置时
   - Then 碰撞音效音量随之改变

3. **快速响应**
   - Given 玩家发生碰撞
   - Then 音效响应延迟 <100ms

4. **死亡流程集成**
   - Given 玩家与障碍物碰撞
   - When 死亡流程触发时
   - Then 碰撞音效在死亡动画/反馈前播放

### 技术验收标准

1. **使用现有 AudioManager**
   - Given AudioManager 单例已存在
   - When 播放碰撞音效时
   - Then 调用 AudioManager.PlaySFX() 方法

2. **音效资源管理**
   - Given 需要播放碰撞音效
   - When 调用 PlaySFX 时
   - Then 使用 AudioClip 资源

3. **与 GameManager 集成**
   - Given GameManager 死亡处理流程
   - When OnPlayerDied 触发时
   - Then 触发碰撞音效播放

4. **代码规范**
   - Given 项目代码规范
   - When 修改现有脚本时
   - Then 遵循命名空间、字段命名、注释规范

## Implementation Guide

### 方案：在 GameManager 的死亡流程中集成

**修改 `GameManager.cs`：**

```csharp
using UnityEngine;
using RunnersJourney.Audio;

namespace RunnersJourney.Game
{
    public class GameManager : MonoBehaviour
    {
        // ... 现有代码 ...

        #region 序列化字段
        [Header("音频")]
        [Tooltip("碰撞音效剪辑")]
        [SerializeField] private AudioClip _collisionSFX;
        #endregion

        // ... 现有代码 ...

        /// <summary>
        /// 玩家死亡处理
        /// </summary>
        private void PlayerDied()
        {
            // 播放碰撞音效（在死亡流程开始时）
            if (_collisionSFX != null && AudioManager.Instance != null)
            {
                AudioManager.Instance.PlaySFX(_collisionSFX);
                if (_enableDebugLog)
                {
                    Debug.Log($"[GameManager] 播放碰撞音效：{_collisionSFX.name}");
                }
            }
            else if (_collisionSFX == null)
            {
                Debug.LogWarning("[GameManager] 碰撞音效未配置，请在 Inspector 中分配 Collision SFX 字段");
            }

            // ... 现有死亡流程代码（停止游戏、重置分数、触发事件等）...
        }
    }
}
```

### 音频资源准备

1. **创建音效文件**
   - 路径：`Assets/Audio/SFX/collision.mp3` 或 `collision.wav`
   - 风格：简短、不刺耳的碰撞/死亡声
   - 时长：0.1-0.3 秒

2. **Unity 导入设置**
   - Force To Mono: ✓（单声道）
   - Load Type: Decompress On Load（低延迟）
   - Preload Audio Data: ✓
