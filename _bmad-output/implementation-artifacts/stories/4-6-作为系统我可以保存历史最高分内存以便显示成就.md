## Acceptance Criteria

### 功能性验收标准

1. **最高分持久化**
   - Given 游戏启动时
   - When ScoreManager 初始化
   - Then 能够读取之前保存的最高分

2. **最高分更新**
   - Given 玩家本次游戏结束
   - When 当前分数超过历史最高分
   - Then 更新最高分并保存

3. **最高分访问**
   - Given 游戏进行中
   - When 需要显示最高分时
   - Then 能够通过 `HighScore` 属性获取

### 技术性验收标准

1. **使用 PlayerPrefs 存储**
   - Given 需要保存最高分
   - When 调用 `SaveHighScore()`
   - Then 使用 `PlayerPrefs.SetInt()` 存储

2. **使用 PlayerPrefs 读取**
   - Given 需要加载最高分
   - When 调用 `LoadHighScore()`
   - Then 使用 `PlayerPrefs.GetInt()` 读取

3. **默认值处理**
   - Given 首次启动游戏
   - When 读取最高分
   - Then 默认值为 0

## Implementation Notes

### Story 4-2 已实现的功能

**ScoreManager.cs** 已经包含以下功能（在 Story 4-2 中实现）：

```csharp
// 最高分属性
public int HighScore { get; private set; }

// 保存最高分
private void SaveHighScore()
{
    PlayerPrefs.SetInt(HighScoreKey, HighScore);
    PlayerPrefs.Save();
}

// 加载最高分
private void LoadHighScore()
{
    HighScore = PlayerPrefs.GetInt(HighScoreKey, 0);
}

// 检查是否打破纪录
public bool CheckHighScore()
{
    if (CurrentScore > HighScore)
    {
        HighScore = CurrentScore;
        SaveHighScore();
        OnScoreChanged?.Invoke(CurrentScore, HighScore);
        return true;
    }
    return false;
}
```
