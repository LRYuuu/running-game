## Acceptance Criteria

### 功能性验收标准

1. **空隙生成概率控制**
   - Given 地图配置中有空隙生成概率配置
   - When 生成新 Chunk 时
   - Then 空隙按配置概率生成（默认 10%）

2. **空隙宽度范围控制**
   - Given 空隙最小宽度=1，最大宽度=3
   - When 生成空隙时
   - Then 空隙宽度在设定范围内随机选择

3. **起始保护区**
   - Given 前 N 个 Chunk 为起始保护区（默认 5 个 Chunk）
   - When 生成地图时
   - Then 起始保护区内不生成空隙

4. **空隙安全性保证**
   - Given 空隙宽度不超过玩家最大跳跃距离
   - When 玩家遇到空隙时
   - Then 玩家可以通过一段跳或二段跳跳过空隙

5. **空隙视觉表现**
   - Given 空隙位置没有地面 Tile
   - When 渲染地图时
   - Then 空隙处显示为空（玩家可以看到背景）

6. **空隙与障碍物互斥**
   - Given 空隙区域
   - When 生成障碍物时
   - Then 空隙上方不生成障碍物

### 技术性验收标准

1. **空隙边界处理**
   - Given 空隙起始列和结束列
   - When 生成地面时
   - Then 空隙左边界使用 grass_right，右边界使用 grass_left

2. **高度缓存处理**
   - Given 空隙列
   - When 更新高度缓存时
   - Then 空隙列高度标记为 0 或特殊值

3. **Chunk 清理时同步清理空隙缓存**
   - Given Chunk 被清理
   - When 清除高度缓存时
   - Then 同步清除空隙范围缓存

### 空隙生成算法设计

#### 1. 空隙生成时机

空隙应该在 Chunk 级别决定，并且**只能在平地（高度相同的连续区域）生成**。这样确保空隙两侧的地面高度一致，玩家可以轻松跳过。

```csharp
// 在 GenerateChunkAtEnd() 中调用
private void GenerateChunkAtEnd()
{
    int startX = ...;

    // 检查是否生成空隙
    bool shouldGenerateGap = ShouldGenerateGap(maxGeneratedChunk, startX);
    int gapWidth = 0;
    int gapStartLocalX = -1;
    int gapGroundHeight = -1;

    if (shouldGenerateGap)
    {
        gapWidth = Random.Range(config.minGapWidth, config.maxGapWidth + 1);
        // 空隙开始位置（Chunk 内部）
        gapStartLocalX = Random.Range(2, config.chunkWidth - gapWidth - 2);

        // 检查空隙位置是否为平地
        int candidateStartX = startX + gapStartLocalX;
        if (!IsFlatGround(candidateStartX, gapWidth, out gapGroundHeight))
        {
            // 不是平地，取消生成空隙
            gapWidth = 0;
            gapStartLocalX = -1;
            shouldGenerateGap = false;
        }
    }

    for (int x = 0; x < config.chunkWidth; x++)
    {
        int worldX = startX + x;

        // 检查是否在空隙范围内
        if (gapStartLocalX >= 0 && x >= gapStartLocalX && x < gapStartLocalX + gapWidth)
        {
            // 空隙列：生成水流 Tile
            GenerateGapColumn(worldX, gapGroundHeight);
            _heightCache[worldX] = 0; // 标记为空隙
            continue;
        }

        // 正常生成地面
        GenerateGroundColumn(worldX, x);

        // 生成障碍物（空隙上方不生成）
        if (gapStartLocalX < 0 && ShouldSpawnObstacle(worldX))
        {
            SpawnObstacle(worldX);
        }
    }
}
```

### 推荐实现方案

基于当前代码结构，推荐以下实现方案：

#### 方案 A：Chunk 级别的空隙生成（推荐）

**优点：**
- 控制简单，每个 Chunk 最多一个空隙
- 容易保证 Chunk 拼接处的空隙连续性
- 性能开销小

**缺点：**
- 空隙位置受 Chunk 边界限制

#### 方案 B：连续的瓦片级别空隙生成

**优点：**
- 空隙位置更灵活
- 可以跨 Chunk 边界

**缺点：**
- 需要维护空隙状态机
- Chunk 拼接处需要特殊处理

**推荐采用方案 A**，因为：
1. 当前代码结构已经是 Chunk 级别的管理
2. 空隙宽度较小（1-3 瓦片），Chunk 级别控制足够
3. 实现复杂度低，风险小

### 高低地块高度差修复（2026-03-17）

**问题描述**：在高低地块交界处生成空隙时，如果空隙左右两侧的地块高度不同，会出现视觉间隙。

**修复方案**：修改空隙生成逻辑，获取空隙**左右两侧**的地块高度，取较低的一侧作为空隙的基准高度。

```csharp
if (shouldGenerateGap)
{
    // 随机选择空隙宽度
    gapWidth = Random.Range(config.minGapWidth, config.maxGapWidth + 1);
    // 空隙开始位置（Chunk 内部，留出边界）
    gapStartLocalX = Random.Range(2, config.chunkWidth - gapWidth - 2);

    // 检查空隙位置的地形高度
    int candidateStartX = startX + gapStartLocalX;
    int candidateEndX = candidateStartX + gapWidth;

    // 获取空隙左右两侧的地块高度
    int leftGroundHeight = GetColumnHeight(candidateStartX - 1);
    int rightGroundHeight = GetColumnHeight(candidateEndX);

    // 检查是否为无效位置（左右两侧有空隙或高度为 0）
    if (leftGroundHeight <= 0 || rightGroundHeight <= 0)
    {
        // 空隙旁边已经是空隙，取消生成
        gapWidth = 0;
        gapStartLocalX = -1;
        shouldGenerateGap = false;
        Debug.Log($"[TilemapMapGenerator] 空隙位置无效（旁边是空隙），取消生成 @ Chunk #{maxGeneratedChunk}");
    }
    else
    {
        // 使用左右两侧较低的地块高度作为空隙基准高度
        gapGroundHeight = Mathf.Min(leftGroundHeight, rightGroundHeight);

        if (leftGroundHeight != rightGroundHeight)
        {
            Debug.Log($"[TilemapMapGenerator] 空隙两侧高度不同 (左={leftGroundHeight}, 右={rightGroundHeight})，使用较低高度 {gapGroundHeight} @ Chunk #{maxGeneratedChunk}");
        }
    }
}
```

**修复效果**：
- 当空隙左右两侧地块高度不同时，使用较低一侧的高度作为空隙的基准高度
- 确保水流顶部与较低的地块齐平，避免视觉间隙
- 同时检测空隙旁边是否已经是空隙，避免连续空隙导致的问题

### 空隙相邻黑边优化（2026-03-18）

**问题描述**：当左右两侧高低不同且有空隙时，高的一侧草坪应该显示黑边（使用 grass_left 或 grass_right），但现有逻辑将空隙相邻判定为"非暴露"，导致高的一侧使用了无缝草块 grass_middle。

**修复方案**：在 `GenerateGroundColumn` 方法中添加 `GetGapOppositeHeight` 辅助方法，跨越空隙获取另一侧的实际地块高度，然后判断当前列是否比另一侧高，如果高则标记为暴露。

```csharp
/// <summary>
/// 获取空隙另一侧的地块高度
/// </summary>
/// <param name="currentX">当前位置</param>
/// <param name="direction">方向：-1=向左查找，1=向右查找</param>
/// <returns>空隙另一侧的地块高度，如果找不到则返回 0</returns>
private int GetGapOppositeHeight(int currentX, int direction)
{
    // 从当前位置向指定方向查找，跳过空隙列
    int searchX = currentX;
    while (IsInGap(searchX))
    {
        searchX += direction;
    }
    // 返回空隙另一侧的地块高度
    return GetColumnHeight(searchX);
}

// 在 GenerateGroundColumn 中
// 4. 获取空隙另一侧的地块高度（如果需要）
int leftGroundHeight = leftHeight;
int rightGroundHeight = rightHeight;

if (leftIsGap)
{
    // 左侧是空隙，获取空隙左侧的地块高度
    leftGroundHeight = GetGapOppositeHeight(worldX, -1);
}

if (rightIsGap)
{
    // 右侧是空隙，获取空隙右侧的地块高度
    rightGroundHeight = GetGapOppositeHeight(worldX, 1);
}

// 5. 判断是否需要黑边（暴露的侧面）
// 普通情况：相邻列比当前列低，且不是空隙
bool leftExposed = (leftHeight < columnHeight) && !leftIsGap;
bool rightExposed = (rightHeight < columnHeight) && !rightIsGap;

// 特殊情况：空隙相邻且当前列比空隙另一侧的地块高
// 此时高的一侧应该显示黑边
if (leftIsGap && columnHeight > rightGroundHeight)
{
    rightExposed = true;
}
if (rightIsGap && columnHeight > leftGroundHeight)
{
    leftExposed = true;
}
```

**修复效果**：
- 左侧地块高度=5，空隙（1 列），右侧地块高度=4
- 左侧地块的右边界会使用 `grass_right`（显示黑边），因为右侧有空隙且左侧更高
- 右侧地块的左边界使用 `grass_middle`（无缝衔接），因为它是较低的一侧
- 视觉效果：高的一侧有明显的边界标识，低的一侧无缝衔接空隙水流

- 远离空隙的一侧（左侧）应该使用 `grassLeft`（显示黑边，因为比左侧高）

// 此时高的一侧应该显示黑边（远离空隙的一侧）
if (leftIsGap && columnHeight > rightGroundHeight)
{
    rightExposed = true;
}
if (rightIsGap && columnHeight > leftGroundHeight)
{
    leftExposed = true;
}

// 特殊情况 2：空隙相邻且高度一致时，靠近空隙的一侧不显示黑边（使用 grassMiddle 无缝衔接）
// 如果有高度差（当前列更高），则保持暴露状态（显示黑边）

{
    rightExposed = false;
}
if (leftIsGap && columnHeight == leftHeight)
{
    leftExposed = false;
}

// 6. 选择草坪 Tile
TileBase grassTile;
if (leftExposed && rightExposed)
{
    // 两侧都暴露：使用 grassIsolated（两侧都有黑边，用于普通单格凸起地块）

    grassTile = config.grassLeft;
}

    grassTile = config.grassRight;
}
else
{

    grassTile = config.grassMiddle;
}
```

**修复效果**：
- 单格凸起地块（5→6→5）旁边有空隙时：

    // 两侧都暴露（单格凸起地块）：土壤层使用普通 dirtTile + 随机翻转
    // 草坪层会使用 grassIsolated 显示两侧黑边，但土壤层不需要
    dirtTileToUse = config.dirtTile;
    dirtFlipMode = GetDirtFlipMode(worldX, 0);
}
else if (leftExposed)
{
    // 仅左侧暴露：使用 dirtLeft（左侧有黑边）
    dirtTileToUse = config.dirtLeft != null ? config.dirtLeft : config.dirtTile;
    dirtFlipMode = 0; // 无翻转，黑边在左侧
}
else if (rightExposed)
{
    // 仅右侧暴露：使用 dirtRight（右侧有黑边）
    dirtTileToUse = config.dirtRight != null ? config.dirtRight : config.dirtTile;
    dirtFlipMode = 0; // 无翻转，黑边在右侧
}
else
{
    // 无暴露（内部地块）：使用 dirtTile，随机翻转
    dirtTileToUse = config.dirtTile;
    dirtFlipMode = GetDirtFlipMode(worldX, 0);
}
```

**修复效果**：
- **单格凸起地块**（5→6→5）：
  - 草坪层：使用 `grassIsolated`（两侧都有黑边）✅
  - 土壤层：使用 `dirtTile`（无黑边，随机翻转）✅
- **内部地块**（5→5→5）：
  - 草坪层：使用 `grassMiddle`（无缝衔接）✅
  - 土壤层：使用 `dirtTile`（无黑边，随机翻转）✅
- **边界地块**（5→[空隙]）：
  - 草坪层：使用 `grassRight`（右侧有黑边）✅
  - 土壤层：使用 `dirtRight`（右侧有黑边）✅
- **视觉效果**：只有真正的边界地块土壤层才显示黑边，内部地块和单格凸起地块的土壤层不显示黑边
