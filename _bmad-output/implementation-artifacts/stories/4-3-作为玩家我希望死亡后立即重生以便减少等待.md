## Story

作为玩家，
我希望死亡后立即自动重生，
以便减少等待时间并保持游戏节奏。

---

## Acceptance Criteria

### 功能性验收标准

1. **死亡延迟配置**
   - Given 玩家碰撞障碍物
   - When 触发死亡
   - Then 等待短暂的死亡动画时间（默认 1 秒）后立即重生

2. **自动重生流程**
   - Given 玩家死亡
   - When 死亡延迟结束
   - Then 自动将玩家重置到检查点位置，无需玩家操作

3. **重生后可立即操作**
   - Given 重生完成
   - When 玩家回到检查点
   - Then 玩家控制器立即恢复，可以正常移动和跳跃

4. **死亡流程不阻塞**
   - Given 死亡动画播放中
   - When 玩家按下跳跃键
   - Then 输入被禁用，不会触发跳跃

### 技术性验收标准

1. **死亡延迟可配置**
   - Given PlayerDeathController 初始化
   - When 设置 respawnDelay 参数
   - Then 使用配置值控制死亡到重生的延迟时间

2. **使用协程实现延迟**
   - Given 死亡流程启动
   - When GameManager 调用 StartRespawn()
   - Then 使用 IEnumerator 和 WaitForSeconds 实现非阻塞延迟

3. **重生流程集中管理**
   - Given 死亡触发
   - When GameManager 切换到 Dying 状态
   - Then 由 GameManager 统一管理重生流程，而不是 PlayerDeathController

4. **与现有系统集成**
   - Given 重生流程触发
   - When 重置玩家位置时
   - Then 同时调用 PlayerJumpController.ResetJumpState() 重置跳跃状态

## Implementation Notes

### 现有代码分析

GameManager.cs 中已有 `StartRespawn()` 协程框架：

```csharp
private IEnumerator StartRespawn()
{
    // 等待死亡延迟
    float respawnDelay = _playerDeathController != null
        ? _playerDeathController.RespawnDelay
        : 1f;

    yield return new WaitForSeconds(respawnDelay);

    ChangeState(GameState.Respawning);

    // 重置玩家状态
    if (_playerJumpController != null)
    {
        _playerJumpController.ResetJumpState();
    }

    // 重置位置
    if (_playerDeathController != null)
    {
        _playerDeathController.SetSafePosition(_lastSafePosition);
        _playerDeathController.RespawnImmediately();
    }

    // 恢复游戏状态
    ChangeState(GameState.Playing);
}
```
