# Story 6-1: 作为系统，我可以生成不同生物群系的地形，提供视觉变化

**Epic:** Epic 6 - 地图生成扩展
**优先级:** P0
**估时:** 4h
**状态:** done

---

## Story

作为系统，
我可以生成不同生物群系的地形，
以便提供视觉变化。

---

## Acceptance Criteria

### 功能验收标准

1. **三种生物群系视觉差异**
   - Given 游戏开始选择群系
   - When 使用不同群系配置生成地图
   - Then 地表 Tile、背景、障碍物呈现对应的视觉主题

2. **群系 Tile 替换**
   - Given 草坪群系使用绿色草地 Tile
   - When 切换到沙漠群系
   - Then 地表 Tile 替换为黄沙 Tile，背景替换为沙漠主题

3. **与地形起伏系统集成**
   - Given 群系切换功能
   - When 地形起伏生成时
   - Then 群系 Tile 替换逻辑与高度变化逻辑协同工作

4. **群系配置可扩展**
   - Given 新的群系需求
   - When 添加新的群系配置资产
   - Then 无需修改代码即可支持新群系

5. **默认群系回退**
   - Given 未指定群系配置
   - When 生成地图时
   - Then 使用默认的草坪群系

6. **支持两种群系模式（独立/混合）**
   - Given 开发者配置 BiomeSequence
   - When 游戏运行时
   - Then 可以选择固定单一群系或随距离自动切换群系

### 技术验收标准

1. **BiomeConfig ScriptableObject**
   - Given 需要配置不同群系的视觉元素
   - When 开发者创建 BiomeConfig 资产
   - Then 可以在 Inspector 中配置地表 Tile、背景材质、颜色等

2. **BiomeSequence 配置（混合模式核心）**
   - Given 需要实现混合群系模式
   - When 开发者创建 BiomeSequence 资产
   - Then 可以配置群系序列和切换距离阈值

3. **地图生成器修改**
   - Given 现有的 TilemapEndlessMapGenerator
   - When 生成 Chunk 时
   - Then 根据当前激活的群系配置选择对应的 Tile

4. **群系切换 API**
   - Given 游戏需要切换群系
   - When 调用 SetBiome() 方法或距离达到阈值
   - Then 后续生成的 Chunk 使用新群系配置

5. **性能要求**
   - Given 群系切换
   - When 生成新 Chunk
   - Then 性能开销不超过原有生成的 10%

### 两种模式对比

| 模式 | 配置方式 | 切换时机 | 使用场景 |
|------|---------|---------|---------|
| **固定模式** | 选择单个 BiomeConfig | 游戏开始时玩家选择/解锁 | 玩家自选群系、特定挑战模式 |
| **混合模式** | 配置 BiomeSequence | 随距离自动切换 | 单局内渐进式体验、第四地图 |

### 生物群系视觉对比

| 元素 | 草坪 (Grassland) | 沙漠 (Desert) | 雪地 (Snowland) |
|------|-----------------|--------------|----------------|
| **地表 Tile** | 翠绿草地 | 金黄沙地 | 雪白地面 |
| **土壤 Tile** | 棕色土壤 | 浅黄土壤 | 冰层土壤 |
| **天空颜色** | 淡蓝色 (#87CEEB) | 橙黄色 (#FFD700) | 冷白色 (#F0F8FF) |
| **背景颜色** | 绿色渐变 | 橙色渐变 | 蓝色渐变 |
| **障碍物** | 石头、树桩 | 仙人掌、岩石 | 冰块、雪球 |

### 混合模式序列示例

```
BiomeSequence: "FullProgression"
├── Stage 1: Grassland (0 - 200 米)
├── Stage 2: Desert (200 - 400 米)
└── Stage 3: Snowland (400+ 米)

距离阈值配置：
- transitionDistance: 200  （群系切换距离）
- 或每个 Stage 独立配置 distanceThreshold
```

### 群系切换流程图（支持两种模式）

```
游戏启动
    │
    ▼
BiomeManager.Initialize()
    │
    ├─ 检查是否使用 BiomeSequence
    │
    ├─ 是 → 加载序列配置
    │   └─ 根据当前距离设置初始群系
    │
    └─ 否 → 使用固定模式
        └─ 使用默认群系/玩家选择的群系
    │
    ▼
游戏进行中（混合模式）
    │
    ├─ GameManager 更新玩家距离
    │
    ├─ BiomeManager.UpdateBiomeByDistance(distance)
    │   │
    │   ├─ 检查是否达到下一个群系阈值
    │   │
    │   ├─ 是 → 切换到新群系
    │   │   ├─ 更新 CurrentBiome
    │   │   └─ 触发 OnBiomeChanged 事件
    │   │
    │   └─ 否 → 保持当前群系
    │
    ▼
TilemapEndlessMapGenerator 生成 Chunk
    │
    ├─ 根据 CurrentBiome 选择 Tile
    │
    └─ 生成对应群系的地形
```

```csharp
// BiomeManager.cs - 支持两种模式的单例
using UnityEngine;
using UnityEngine.Events;

namespace RunnersJourney.Map
{
    public class BiomeManager : MonoBehaviour
    {
        private static BiomeManager _instance;
        public static BiomeManager Instance
        {
            get
            {
                if (_instance == null)
                {
                    _instance = FindObjectOfType<BiomeManager>();
                }
                return _instance;
            }
        }

        [Header("默认群系")]
        [Tooltip("游戏启动时使用的默认群系（固定模式）")]
        public BiomeConfig defaultBiome;

        [Header("群系序列（混合模式）")]
        [Tooltip("群系序列配置（可选，用于混合模式）")]
        public BiomeSequence biomeSequence;

        // 当前激活的群系
        public BiomeConfig CurrentBiome { get; private set; }

        // 当前使用的序列（混合模式）
        public BiomeSequence CurrentSequence { get; private set; }

        // 当前距离（混合模式）
        private float _currentDistance = 0f;

        // 当前群系阶段索引（混合模式）
        private int _currentStageIndex = 0;

        // 群系切换事件
        public UnityEvent<BiomeConfig> OnBiomeChanged = new();

        private void Awake()
        {
            if (_instance != null && _instance != this)
            {
                Destroy(gameObject);
                return;
            }
            _instance = this;
            DontDestroyOnLoad(gameObject);
        }

        private void Start()
        {
            // 优先使用序列配置（混合模式）
            if (biomeSequence != null && biomeSequence.useSequence)
            {
                SetBiomeSequence(biomeSequence, 0f);
            }
            // 否则使用固定模式
            else if (defaultBiome != null)
            {
                SetBiome(defaultBiome);
            }
        }

        /// <summary>
        /// 固定模式：设置单一群系
        /// </summary>
        public void SetBiome(BiomeConfig newBiome)
        {
            if (newBiome == null)
            {
                Debug.LogWarning("[BiomeManager] 尝试设置为 null 的群系，使用默认群系");
                newBiome = defaultBiome;
            }

            CurrentBiome = newBiome;
            CurrentSequence = null;
            Debug.Log($"[BiomeManager] 切换到群系：{newBiome.biomeName}（固定模式）");
            OnBiomeChanged?.Invoke(newBiome);
        }

        /// <summary>
        /// 混合模式：设置群系序列
        /// </summary>
        public void SetBiomeSequence(BiomeSequence sequence, float distance = 0f)
        {
            if (sequence == null)
            {
                Debug.LogWarning("[BiomeManager] 序列配置为 null");
                return;
            }

            CurrentSequence = sequence;
            _currentDistance = distance;
            _currentStageIndex = 0;

            // 根据当前距离设置初始群系
            UpdateBiomeByDistance(distance);
        }

        /// <summary>
        /// 混合模式：根据距离更新群系
        /// </summary>
        public void UpdateBiomeByDistance(float distance)
        {
            _currentDistance = distance;

            if (CurrentSequence == null || !CurrentSequence.useSequence)
            {
                return; // 固定模式，不需要更新
            }

            // 查找当前距离对应的群系阶段
            int newStageIndex = 0;
            float accumulatedDistance = 0f;

            for (int i = 0; i < CurrentSequence.biomeStages.Count; i++)
            {
                var stage = CurrentSequence.biomeStages[i];

                if (stage.biome == null)
                {
                    Debug.LogWarning($"[BiomeManager] 阶段 {i} 的群系配置为空，跳过");
                    continue;
                }

                // 检查是否达到切换距离
                if (distance >= accumulatedDistance + stage.transitionDistance)
                {
                    accumulatedDistance += stage.transitionDistance;
                    newStageIndex = i + 1;
                }
                else
                {
                    // 找到当前阶段
                    if (i != _currentStageIndex)
                    {
                        // 切换群系
                        CurrentBiome = stage.biome;
                        _currentStageIndex = i;
                        Debug.Log($"[BiomeManager] 切换到群系：{stage.biome.biomeName}（距离：{distance} 米，阶段：{i + 1}/{CurrentSequence.biomeStages.Count}）");
                        OnBiomeChanged?.Invoke(CurrentBiome);
                    }
                    return;
                }
            }

            // 如果超过所有阶段，使用最后一个群系
            if (newStageIndex >= CurrentSequence.biomeStages.Count)
            {
                var lastStage = CurrentSequence.biomeStages[CurrentSequence.biomeStages.Count - 1];
                if (CurrentBiome != lastStage.biome)
                {
                    CurrentBiome = lastStage.biome;
                    _currentStageIndex = CurrentSequence.biomeStages.Count - 1;
                    Debug.Log($"[BiomeManager] 达到最终群系：{lastStage.biome.biomeName}（距离：{distance} 米）");
                    OnBiomeChanged?.Invoke(CurrentBiome);
                }
            }
        }

        /// <summary>
        /// 获取当前群系（供地图生成器使用）
        /// </summary>
        public BiomeConfig GetCurrentBiome()
        {
            return CurrentBiome ?? defaultBiome;
        }

        /// <summary>
        /// 重置进度（死亡重生时调用）
        /// </summary>
        public void ResetProgress()
        {
            _currentDistance = 0f;
            _currentStageIndex = 0;

            if (CurrentSequence != null && CurrentSequence.useSequence)
            {
                // 混合模式：重置到起始群系
                UpdateBiomeByDistance(0f);
            }
            else if (defaultBiome != null)
            {
                // 固定模式：重置到默认群系
                SetBiome(defaultBiome);
            }
        }
    }
}
```

```csharp
// TilemapEndlessMapGenerator.cs (修改部分)
using UnityEngine;
using UnityEngine.Tilemaps;

namespace RunnersJourney.Map
{
    public class TilemapEndlessMapGenerator : MonoBehaviour
    {
        [Header("生物群系")]
        [Tooltip("生物群系管理器")]
        private BiomeManager _biomeManager;

        [Header("默认 Tile（回退用）")]
        [Tooltip("默认地表 Tile")]
        public TileBase defaultGroundTile;

        [Tooltip("默认土壤 Tile")]
        public TileBase defaultDirtTile;

        private void Awake()
        {
            _biomeManager = BiomeManager.Instance;
            if (_biomeManager != null)
            {
                _biomeManager.OnBiomeChanged.AddListener(OnBiomeChanged);
            }
        }

        private void OnDestroy()
        {
            if (_biomeManager != null)
            {
                _biomeManager.OnBiomeChanged.RemoveListener(OnBiomeChanged);
            }
        }

        /// <summary>
        /// 群系切换时的回调
        /// </summary>
        private void OnBiomeChanged(BiomeConfig newBiome)
        {
            Debug.Log($"[MapGenerator] 群系已切换为 {newBiome.biomeName}，后续 Chunk 将使用新配置");
            // 可选：触发已生成 Chunk 的 Tile 更新
        }

        /// <summary>
        /// 获取当前 Chunk 应该使用的群系
        /// </summary>
        private BiomeConfig GetBiomeForChunk(int chunkX)
        {
            if (_biomeManager != null)
            {
                return _biomeManager.GetCurrentBiome();
            }
            return null;
        }

        private TileBase GetGroundTile()
        {
            var biome = GetBiomeForChunk(0);
            if (biome != null && biome.groundTile != null)
            {
                return biome.groundTile;
            }
            // 回退到默认配置
            return defaultGroundTile;
        }

        private TileBase GetDirtTile()
        {
            var biome = GetBiomeForChunk(0);
            if (biome != null && biome.dirtTile != null)
            {
                return biome.dirtTile;
            }
            // 回退到默认配置
            return defaultDirtTile;
        }

        private GameObject[] GetObstaclePrefabs()
        {
            var biome = GetBiomeForChunk(0);
            if (biome != null && biome.obstaclePrefabs != null)
            {
                return biome.obstaclePrefabs;
            }
            // 回退到默认配置
            return null;
        }
    }
}
```

```csharp
// GameManager.cs (添加距离追踪和群系更新)
using UnityEngine;

namespace RunnersJourney.Game
{
    public class GameManager : MonoBehaviour
    {
        [Header("生物群系（混合模式）")]
        [Tooltip("是否启用混合模式（随距离自动切换群系）")]
        public bool useBiomeSequence = false;

        private float _playerDistance;
        private BiomeManager _biomeManager;

        private void Awake()
        {
            _biomeManager = BiomeManager.Instance;
        }

        private void Update()
        {
            if (currentState == GameState.Playing)
            {
                // 更新玩家距离
                _playerDistance += playerController.velocity.x * Time.deltaTime;

                // 混合模式：更新群系
                if (useBiomeSequence && _biomeManager != null)
                {
                    _biomeManager.UpdateBiomeByDistance(_playerDistance);
                }
            }
        }

        private void OnPlayerDeath()
        {
            // 重置距离和群系进度
            _playerDistance = 0f;
            if (_biomeManager != null)
            {
                _biomeManager.ResetProgress();
            }
        }

        public float GetPlayerDistance() => _playerDistance;
    }
}
```

## Implementation Notes

### 实现建议

1. **BiomeConfig 设计**
   - 使用 ScriptableObject 便于策划在 Inspector 中调整
   - 预留扩展字段（如音效、粒子效果）
   - 使用 `[Range]` 属性限制数值范围

2. **BiomeSequence 设计（混合模式核心）**
   - 使用 `List<BiomeStage>` 配置群系序列
   - 每个阶段配置 `biome` 和 `transitionDistance`
   - `useSequence` 开关控制是否启用混合模式

3. **Tile 替换策略**
   - 在 `GenerateChunk()` 中根据当前群系选择 Tile
   - 保持原有地形起伏逻辑不变
   - 仅替换 Tile 引用，不修改生成算法

4. **群系切换时机**
   - **固定模式**：在游戏开始时选择群系
   - **混合模式**：随距离增加自动切换（默认 200 米一切换）

5. **调试支持**
   - 在 Console 中输出群系切换日志
   - 可在 UI 上显示当前群系名称和距离

6. **性能优化**
   - BiomeManager 使用单例模式
   - 群系配置资产预加载到内存
   - Tile 引用缓存避免重复查找

### Tile 资源需求

参考 `docs/game-design/TileRequirements.md`:

**草坪群系**:
- 草地 Tile × 3（不同视觉变化）
- 土壤 Tile × 1（通过翻转创造变化）

**沙漠群系**:
- 沙地 Tile × 3
- 沙土 Tile × 1

**雪地群系**:
- 雪地 Tile × 3
- 冰层 Tile × 1

### 与现有系统集成

1. **地形起伏系统**
   - 群系切换不影响高度计算逻辑
   - Perlin Noise 生成的高度值决定 Tile 放置位置
   - 群系决定使用哪个 Tile 资源

2. **难度系统**
   - 难度系统（DifficultyCalculator）与群系系统独立
   - 两者可同时影响地图生成结果

3. **GameManager**
   - **固定模式**：GameManager 调用 `BiomeManager.SetBiome()` 选择群系
   - **混合模式**：GameManager 在 `Update()` 中调用 `BiomeManager.UpdateBiomeByDistance()`
   - 群系解锁状态可由 GameManager 跟踪

### 两种模式的使用说明

**固定模式（推荐用于玩家自选）**：
```csharp
// 玩家在主界面选择群系后调用
biomeManager.SetBiome(selectedBiome);
```

**混合模式（推荐用于渐进式体验）**：
```csharp
// 游戏启动时设置序列
biomeManager.SetBiomeSequence(fullSequence, 0f);

// 游戏进行中每帧更新
void Update() {
    float distance = GetPlayerDistance();
    biomeManager.UpdateBiomeByDistance(distance);
}
```

### 混合模式距离阈值建议

```
FullSequence.asset 配置示例：
├── Stage 1: Grassland → Desert @ 200 米
├── Stage 2: Desert → Snowland @ 200 米（累计 400 米）
└── Stage 3: Snowland (最终阶段，无限延伸)

调优建议：
- 前期（0-200 米）：草坪，让玩家熟悉操作
- 中期（200-400 米）：沙漠，增加视觉新鲜感
- 后期（400+ 米）：雪地，高难度挑战区域
```
