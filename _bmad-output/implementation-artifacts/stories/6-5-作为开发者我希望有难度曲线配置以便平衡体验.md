## Acceptance Criteria

### 功能验收标准

1. **难度配置可视化**
   - Given 开发者在 Unity Inspector 中查看 DifficultyConfig
   - When 调整难度参数时
   - Then 可以直观地看到所有配置选项和当前值

2. **难度曲线可配置**
   - Given 四种难度曲线类型（线性、指数、对数、自定义）
   - When 开发者选择不同的曲线类型
   - Then 可以预览或调整对应的曲线效果

3. **配置参数完整**
   - Given 难度系统需要调优
   - When 开发者在 Inspector 中配置
   - Then 可以调整所有必要的参数（起点、终点、概率范围等）

4. **默认配置合理**
   - Given 新创建的 DifficultyConfig 资产
   - When 使用默认值
   - Then 游戏体验平衡（不太简单也不太困难）

5. **配置验证**
   - Given 开发者配置了不合理的参数
   - When 参数值超出有效范围
   - Then 有 Range 属性限制或警告提示

### 技术验收标准

1. **ScriptableObject 规范**
   - Given DifficultyConfig 使用 [CreateAssetMenu] 创建
   - When 开发者在项目中创建配置
   - Then 遵循项目命名和目录规范

2. **配置参数有 Tooltip**
   - Given 每个公开字段
   - When 开发者悬停查看
   - Then 有清晰的 Tooltip 说明用途

3. **配置验证逻辑**
   - Given 配置中的数值关系
   - When startDistance > maxDistance 等无效情况
   - Then 在 Awake 或 Start 中记录警告

4. **调试支持**
   - Given 开发模式下运行
   - When 需要调优难度参数
   - Then 可以在控制台或使用工具查看难度状态

### 难度曲线类型

```
难度值 (0.0 - 1.0)
    ^
1.0 |                               ┌─────── 最大难度
    |                          ┌────┘
    |                      ┌───┘
0.8 |                  ┌───┘
    |              ┌───┘
0.6 |          ┌───┘
    |      ┌───┘
0.4 |  ┌───┘
    |  │
0.2 |  │ 安全区域
    |  │ (前 startDistance 米)
0.0 └──┴──────────────────────────────────────> 距离
    0  startDistance          maxDistance

曲线类型：
- Linear（线性）：均匀增长，适合大多数游戏
- Exponential（指数）：先慢后快，适合 Roguelike
- Logarithmic（对数）：先快后慢，适合休闲游戏
- Custom（自定义）：使用 AnimationCurve 完全可控
```

### 配置参数说明

```csharp
// DifficultyConfig 字段说明
public class DifficultyConfig : ScriptableObject
{
    [Header("难度起点")]
    [Tooltip("开始增加难度的距离，此距离内保持最低难度")]
    public float startDistance = 50f;

    [Header("难度顶点")]
    [Tooltip("达到最大难度的距离，超过此距离后难度不再增加")]
    public float maxDistance = 500f;

    [Header("障碍概率")]
    [Tooltip("基础障碍概率（最低难度时的概率）")]
    [Range(0f, 1f)]
    public float baseObstacleChance = 0.3f;

    [Tooltip("最大障碍概率（最高难度时的概率）")]
    [Range(0f, 1f)]
    public float maxObstacleChance = 0.6f;

    [Header("空隙概率")]
    [Tooltip("基础空隙概率（最低难度时的概率）")]
    [Range(0f, 1f)]
    public float baseGapChance = 0.1f;

    [Tooltip("最大空隙概率（最高难度时的概率）")]
    [Range(0f, 1f)]
    public float maxGapChance = 0.25f;

    [Header("曲线类型")]
    [Tooltip("难度增长曲线类型")]
    public CurveType curveType = CurveType.Linear;

    [Header("自定义曲线")]
    [Tooltip("自定义难度曲线 - 仅当 curveType=Custom 时使用")]
    public AnimationCurve customCurve = AnimationCurve.Linear(0f, 0f, 1f, 1f);
}
```

### 配置验证逻辑

```csharp
// 在 DifficultyConfig.cs 中添加
public void ValidateConfiguration()
{
    bool hasWarning = false;

    if (startDistance >= maxDistance)
    {
        Debug.LogWarning($"[DifficultyConfig] startDistance ({startDistance}) 应该小于 maxDistance ({maxDistance})");
        hasWarning = true;
    }

    if (baseObstacleChance > maxObstacleChance)
    {
        Debug.LogWarning($"[DifficultyConfig] baseObstacleChance ({baseObstacleChance}) 不应该大于 maxObstacleChance ({maxObstacleChance})");
        hasWarning = true;
    }

    if (baseGapChance > maxGapChance)
    {
        Debug.LogWarning($"[DifficultyConfig] baseGapChance ({baseGapChance}) 不应该大于 maxGapChance ({maxGapChance})");
        hasWarning = true;
    }

    if (!hasWarning)
    {
        Debug.Log("[DifficultyConfig] 配置验证通过 ✓");
    }
}
```

### 配置调优指南

#### 推荐配置值

| 游戏类型 | startDistance | maxDistance | baseObstacle | maxObstacle | baseGap | maxGap | 曲线类型 |
|----------|---------------|-------------|--------------|-------------|---------|--------|----------|
| 休闲模式 | 100 | 300 | 0.2 | 0.4 | 0.05 | 0.15 | Linear |
| 标准模式 | 50 | 500 | 0.3 | 0.6 | 0.1 | 0.25 | Linear |
| 挑战模式 | 30 | 400 | 0.35 | 0.7 | 0.15 | 0.3 | Exponential |
| Roguelike | 20 | 600 | 0.25 | 0.8 | 0.1 | 0.35 | Exponential |

#### 调优步骤

1. **确定目标受众**
   - 休闲玩家 → 使用休闲模式配置
   - 核心玩家 → 使用标准或挑战模式

2. **调整起点和终点**
   - startDistance: 给新手足够的熟悉距离
   - maxDistance: 决定游戏节奏

3. **调整概率范围**
   - 障碍概率影响躲避频率
   - 空隙概率影响跳跃挑战

4. **选择曲线类型**
   - Linear: 最平衡，适合测试基准
   - Exponential: 后期难度陡增
   - Logarithmic: 早期难度快速提升
