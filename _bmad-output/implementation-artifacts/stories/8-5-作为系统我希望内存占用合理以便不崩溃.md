## Acceptance Criteria

### 功能验收标准

1. **内存占用稳定**
   - Given 游戏长时间运行
   - When 持续游戏 30 分钟以上
   - Then 内存占用保持稳定，无持续增长趋势

2. **无内存泄漏**
   - Given 游戏运行状态
   - When 反复死亡重生
   - Then 内存使用不会持续增加

3. **缓存管理有效**
   - Given 地图生成系统
   - When 清理远离玩家的 Chunk 时
   - Then 相关缓存数据同时被清理

4. **内存监控可见**
   - Given 开发者需要调试内存
   - When 查看性能数据时
   - Then 能够获取内存占用量等关键指标

## Implementation Guide

### 验证流程

**Step 1: 内存基准测试**

使用 Unity Profiler 进行内存分析：
1. 打开 Profiler (Window > Analysis > Profiler)
2. 选择 Memory 模块
3. 运行游戏 5-10 分钟
4. 记录内存使用情况

**Step 2: 缓存验证**

验证 `_heightCache` 内存管理：
```csharp
// 在 TilemapEndlessMapGenerator.cs 中检查：
// 1. ClearHeightCache 是否被正确调用
// 2. _heightCache.Count 是否随游戏时间增长
```

**Step 3: 资源检查**

检查关键资源内存占用：
- Tile 资源：是否被重复加载
- 音频资源：AudioClip 是否被正确释放
- UI 元素：VisualElement 是否有泄漏

**Step 4: 长时间测试**

运行游戏 30 分钟以上，监控：
- 内存占用是否稳定
- GC 触发频率
- 是否有内存峰值

### 可能的优化点

1. **_heightCache 优化**
   - 添加缓存大小限制
   - 实现 LRU 淘汰策略（如需要）

2. **资源卸载**
   - 在场景切换时调用 `Resources.UnloadUnusedAssets()`
   - 考虑使用 Addressables 进行资源管理

3. **对象池扩展**
   - 如发现高频创建对象，考虑对象池化
