**Epic:** Epic 6 - 地图生成扩展

## Story

作为玩家，
我希望不同群系有特殊元素，
以便有新鲜感。

---

## Acceptance Criteria

### 功能验收标准

1. **草地群系特殊元素**
   - Given 选择草地群系
   - When 游戏运行时
   - Then 可以看到绿色草地 Tile、花朵装饰等自然元素

2. **沙漠群系特殊元素**
   - Given 选择沙漠群系
   - When 游戏运行时
   - Then 可以看到黄沙 Tile、仙人掌障碍物、石头等特色元素

3. **雪地群系特殊元素**
   - Given 选择雪地群系
   - When 游戏运行时
   - Then 可以看到白雪 Tile、冰块障碍物、雪人等特色元素

4. **群系特殊障碍物**
   - Given 不同群系有不同的障碍物配置
   - When 生成障碍物时
   - Then 使用对应群系特有的障碍物 Tile

5. **群系背景差异**
   - Given 不同群系
   - When 切换群系时
   - Then 背景颜色和天空颜色随之改变

### 技术验收标准

1. **BiomeConfig 扩展**
   - Given BiomeConfig 已有 obstacleTiles 字段
   - When 配置不同群系时
   - Then 可以为每个群系配置独特的障碍物 Tile 列表

2. **地图生成器支持**
   - Given TilemapEndlessMapGenerator 已集成 BiomeManager
   - When 生成障碍物时
   - Then 从当前群系的 obstacleTiles 中随机选择

3. **配置验证**
   - Given 群系未配置障碍物
   - When 生成障碍物时
   - Then 使用默认障碍物作为回退

4. **性能无影响**
   - Given 使用群系特殊元素
   - When 生成地图时
   - Then 性能开销不超过原有生成的 5%

### 障碍物生成数据流

```
生成 Chunk
    │
    ▼
ShouldSpawnObstacle() 检查是否生成
    │
    ├── Yes ──> GetObstaclePrefab()
    │              │
    │              ├─> 检查 CurrentBiome.obstaclePrefabs
    │              │
    │              ├─> 如果为空，使用 defaultObstacleTiles
    │              │
    │              └─> 随机选择一个障碍物预制体
    │
    └── No ──> 不生成障碍物

Instantiate(ObstaclePrefab, position, rotation)
    │
    ▼
添加到 obstacleTilemap 或作为独立 GameObject
```

### 群系切换流程

```
GameManager.StartGame()
    │
    ▼
ApplyBiomeSelection()
    │
    ├─> 检查玩家选择（固定模式/混合模式）
    │
    ▼
BiomeManager.SetBiome() 或 SetBiomeSequence()
    │
    ▼
OnBiomeChanged 事件触发
    │
    ├──────> TilemapEndlessMapGenerator
    │         └─> 更新当前使用的 obstaclePrefabs
    │
    └──────> CameraController (可选)
              └─> 更新 backgroundColor 和 skyColor
```

### 默认障碍物配置

如果群系未配置 obstaclePrefabs，使用以下回退逻辑：

```csharp
// TilemapEndlessMapGenerator.cs
private GameObject GetObstaclePrefab()
{
    // 优先使用当前群系的障碍物
    if (_biomeManager != null &&
        _biomeManager.CurrentBiome != null &&
        _biomeManager.CurrentBiome.obstaclePrefabs != null &&
        _biomeManager.CurrentBiome.obstaclePrefabs.Length > 0)
    {
        int randomIndex = Random.Range(0, _biomeManager.CurrentBiome.obstaclePrefabs.Length);
        return _biomeManager.CurrentBiome.obstaclePrefabs[randomIndex];
    }

    // 回退到默认障碍物
    if (defaultObstacleTiles != null && defaultObstacleTiles.Length > 0)
    {
        // 返回一个使用默认 Tile 的障碍物
        // ...
    }

    return null;
}
```

## Implementation Notes

### 实现建议

1. **优先级：配置优先于代码**
   - 首先检查三个 BiomeConfig 资产的配置
   - 如果 obstaclePrefabs 为空，可以：
     - 在 Inspector 中手动配置（推荐，无需代码修改）
     - 或创建简单的占位障碍物预制体

2. **障碍物命名规范**
   - 使用 `Obs_` 前缀：`Obs_Tree_Grass`, `Obs_Cactus`, `Obs_IceBlock`
   - 放在 `Assets/Prefabs/Obstacles/` 目录下

3. **障碍物组件要求**
   - `SpriteRenderer`: 显示障碍物图像
   - `BoxCollider2D` 或 `CircleCollider2D`: 碰撞检测
   - 可选：`Rigidbody2D`（Kinematic，用于物理交互）

4. **背景颜色更新**
   - 可以在 BiomeManager 中添加事件监听
   - 或者在 GameManager.ApplyBiomeSelection() 中直接设置

### 配置检查清单

### 与现有系统集成

1. **TilemapEndlessMapGenerator**
   - 已有 `_biomeManager` 引用
   - 已有 `config` 包含默认障碍概率
   - 需要修改 `ShouldSpawnObstacle()` 方法从群系配置获取障碍物

2. **BiomeManager**
   - 已有 `OnBiomeChanged` 事件
   - 已有 `CurrentBiome` 属性
   - 可选：添加 `ApplyBackground()` 方法

3. **GameManager**
   - 已有 `ApplyBiomeSelection()` 方法
   - 无需修改

### 性能考虑

1. **对象池（可选优化）**
   - 如果障碍物频繁生成/销毁，考虑使用对象池
   - 对于本项目（障碍物与地图一起滚动并清理），对象池不是必需的

2. **SpriteAtlas**
   - 将所有障碍物 Sprite 打包到一个 SpriteAtlas
   - 减少 Draw Calls，提高渲染性能
