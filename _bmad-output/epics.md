## Epic 1: 核心玩家控制器

### Goal

实现玩家的核心控制能力，包括一段跳、二段跳和物理交互，确保操作手感流畅。

### Scope

**Includes:**
- 玩家 GameObject 设置（Sprite、Rigidbody2D、Collider2D）
- 地面检测系统（判断是否可执行一段跳）
- 一段跳实现（固定高度）
- 二段跳实现（空中任意时刻可触发）
- 跳跃手感调优（响应时间 <100ms）
- 落地缓冲期实现（允许连续跳跃）
- 土狼时间（Coyote Time，可选）

**Excludes:**
- 移动控制（自动奔跑设计，无左右控制）
- 动画系统（Epic 8）
- 粒子效果（Epic 8）

### Dependencies

- 无（可并行于 Epic 2 启动）

### Deliverable

一个可控制跳跃的玩家角色，手感调优完成，可与其他系统（地图、障碍物）集成测试。

### Stories

| ID | 故事 | 估算 |
|----|------|------|
| 1.1 | 作为玩家，我可以按跳跃键起跳，以便越过障碍 | 2h |
| 1.2 | 作为玩家，我可以在空中再次跳跃（二段跳），以便修正失误 | 2h |
| 1.3 | 作为玩家，我希望跳跃响应迅速，以便精确控制时机 | 1h |
| 1.4 | 作为玩家，我希望落地后可以立即再次跳跃，以便保持节奏 | 1h |
| 1.5 | 作为玩家，我希望走出平台边缘后有短暂跳跃窗口，以便提升手感（土狼时间） | 2h |
| 1.6 | 作为开发者，我希望有地面检测系统，以便正确判断一段跳条件 | 2h |
| 1.7 | 作为开发者，我希望有跳跃参数配置，以便快速调优手感 | 1h |

## Epic 2: 基础地图系统

### Goal

实现程序化地图生成系统，支持无尽地形生成和动态 Chunk 管理。

### Scope

**Includes:**
- Tilemap 系统设置
- Chunk 预制体设计（20×5 瓦片）
- 程序化生成算法（基于种子）
- 动态 Chunk 生成（玩家前进时生成新 Chunk）
- 动态 Chunk 清理（远离玩家时清理旧 Chunk）
- 基础地形生成（平坦地面）
- Perlin Noise 地形起伏算法
- 相邻 Chunk 高度差限制（≤1）
- 起始保护区（前 3 Chunk 平坦）

**Excludes:**
- 生物群系系统（Epic 6）
- 障碍物生成（Epic 3）
- 空隙生成（Epic 3）

### Dependencies

- 无（可并行于 Epic 1 启动）

### Deliverable

一个可以无尽生成的地图系统，地形有自然起伏，性能优化良好。

### Stories

| ID | 故事 | 估算 |
|----|------|------|
| 2.1 | 作为系统，我可以按 Chunk 为单位生成地形，以便优化性能 | 3h |
| 2.2 | 作为系统，我可以动态清理远离玩家的 Chunk，以便节省内存 | 2h |
| 2.3 | 作为玩家，我希望地形有自然起伏，以便游戏体验更丰富 | 3h |
| 2.4 | 作为玩家，我希望相邻地形高度差不会太大，以便可以跳过 | 2h |
| 2.5 | 作为玩家，我希望游戏开始时有一段平坦地形，以便熟悉操作 | 1h |
| 2.6 | 作为开发者，我希望使用种子生成地图，以便可复现测试 | 2h |
| 2.7 | 作为开发者，我希望有地图配置 ScriptableObject，以便调整参数 | 2h |
| 2.8 | 作为开发者，我希望 Tile 可以有翻转变化，以便视觉更丰富 | 2h |

- 障碍物生成算法（概率控制）

- 玩家与地形碰撞检测
- 死亡触发逻辑

**Excludes:**
- 动态障碍物（移动平台、飞鸟等，P1）
