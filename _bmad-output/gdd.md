# Square Fireline - Game Design Document

**Author:** liruoyu

**Target Platform(s):** 全平台（PC、移动端 iOS/Android、主机）

## Executive Summary

### Game Name

Square Fireline

### Core Concept

Square Fireline 是一款通过程序化生成提供无限重玩价值的 2D 横板跳跃游戏。项目采用 Unity 2022.3.62f3 LTS 开发，核心特色包括：

- **程序化地图生成系统**：基于 Tilemap 的无尽地图生成，每次游戏体验都是独特的
- **Chunk 动态生成**：20×5 瓦片为单位，动态生成/清理，性能优化
- **Perlin Noise 地形起伏**：创造自然流畅的地形变化，相邻 Chunk 高度差限制≤1
- **Tile 翻转技术**：单个土壤 Tile 通过 4 种翻转创造视觉变化，优化资源使用

游戏采用作品集导向开发策略，重点展示 AI 辅助开发工作流和 BMad 方法的实践应用。

### Game Type

**Type:** Action Platformer（动作平台跳跃）
**Framework:** 本 GDD 使用 action-platformer 模板，包含类型特定章节：移动系统、平台跳跃挑战生成、障碍物设计

### Target Audience

**目标用户:** 休闲游戏玩家 - 追求轻松、易上手的游戏体验

**设计理念:** 简单易上手 + 技术展示价值（作品集导向）

### Unique Selling Points (USPs)

| 差异化因素 | 说明 |

| **地形起伏系统** | 使用 Perlin Noise 算法创造自然流畅的地形变化 |
| **Tile 翻转技术** | 单个 Tile 通过 4 种翻转创造视觉变化，优化资源使用 |
| **动态 Chunk 系统** | 按 Chunk 为单位动态生成/清理，性能优化，内存占用低 |

## Core Gameplay

### Game Pillars

| 支柱 | 说明 |
|------|------|
| **简单易上手** | 30 秒内理解操作，1 分钟内上手，无需冗长教程 |
| **流畅操作** | 输入响应<100ms，稳定 60FPS，二段跳手感调优为核心 |
| **无限重玩价值** | 程序化地图生成，每次游戏体验都是独特的 |
| **快速重试** | 死亡后立即重生，无等待，保持游戏节奏 |

**支柱优先级：** 当支柱冲突时，按以下顺序优先：
1. 流畅操作（操作手感是平台跳跃的核心）
2. 简单易上手（休闲玩家友好）
3. 快速重试（减少挫败感）
4. 无限重玩价值（长期吸引力）

---

### Core Gameplay Loop

**循环描述：**

```
启动游戏 → 奔跑 → 跳跃 → 躲避障碍 → (重复) → 死亡 → 重生 → 重复
```

**循环图示：**
```
     ┌─────────────────────────────────────┐
     │                                     ▼
[开始游戏] → [奔跑] → [跳跃] → [躲避障碍] → [死亡？]
                                     │
                           ┌─────否────┘
                           ▼
                      [继续游戏] → [分数增加]
                           │
                           是
                           ▼
                    [重生] ──┘
```

**循环时长：**
- 单次循环（跳跃到下一次跳跃）：2-5 秒
- 完整循环（开始到死亡）：30 秒 -3 分钟（依技能水平）

**循环变化：**
- 程序化生成的地图确保每次地形不同
- 障碍物类型和位置变化
- 地形起伏增加挑战性

### Win/Loss Conditions

#### Victory Conditions

**成功是持续性的** - 无尽跑酷模式：
- 没有传统意义上的"通关"
- 成功定义为：创造个人最高分
- 分数基于生存时间/躲避障碍数量

#### Failure Conditions

| 失败条件 | 触发方式 |
|---------|---------|
| **触碰障碍物** | 玩家与障碍物（石头、尖刺等）碰撞 |
| **生命值耗尽** | 撞击导致的生命值减少至 0（如实现） |
| **掉落空隙** | 未能跳过地形间隙（如实现） |

#### Failure Recovery

**死亡后处理：**
1. 立即触发重生（无等待时间）
2. 玩家重生到当前检查点位置
3. 分数重置为 0（当前运行）
4. 历史最高分保留
5. 地图种子不变，维持地形连续性

## Game Mechanics

### Primary Mechanics

| 机制 | 说明 | 服务支柱 |
|------|------|---------|
| **跳跃 (Jump)** | 核心机制 - 按一次键执行一段跳，在空中再按一次键执行二段跳 | 流畅操作、简单易上手 |
| **自动奔跑 (Auto-Run)** | 角色自动向右奔跑，玩家无需控制移动 | 简单易上手 |
| **躲避 (Dodge)** | 通过跳跃避开障碍物（石头、尖刺、空隙） | 简单易上手 |
| **重生 (Respawn)** | 死亡后立即在当前检查点重生，无等待时间 | 快速重试 |

### Mechanic Interactions

**跳跃 + 自动奔跑**：玩家在自动奔跑中专注跳跃时机，简化操作
**二段跳 + 空隙**：二段跳用于跨越更大的空隙或修正失误
**连续跳跃**：落地后立即再次跳跃，保持节奏流畅

### Mechanic Progression

- **初始状态**：一段跳、二段跳、自动奔跑
- **精通路径**：掌握二段跳时机、连续跳跃节奏、障碍物预判
- **无解锁系统**：所有机制初始可用，难度通过地图生成和障碍物密度调整

## Controls and Input

### Control Scheme (全平台)

**单按钮设计** - 跳跃是唯一输入：

| 平台 | 跳跃控制 |
|------|---------|
| **PC (键盘)** | 空格键 (Space) - 按一次跳跃，空中再按一次二段跳 |
| **移动端 (触屏)** | 点击屏幕任意位置 - 按一次跳跃，空中再按一次二段跳 |
| **主机 (手柄)** | A 按钮 (Xbox) / X 按钮 (PlayStation) - 按一次跳跃，空中再按一次二段跳 |

### Input Feel

**跳跃手感要求：**
- **响应时间**：输入延迟 <100ms
- **一段跳**：地面按跳跃键立即起跳
- **二段跳**：空中按跳跃键立即触发，无严格时机限制
- **落地缓冲**：落地后有短暂缓冲期，允许连续跳跃

### Accessibility Controls

**计划中的无障碍选项：**
- **单指/单键模式**：跳跃为唯一必需输入（已实现）
- **触摸区域调整**：移动端可自定义点击敏感区域
- **视觉辅助**：色盲友好的障碍物高亮选项

## Action Platformer Specific Design

### Movement System

**核心移动能力：**

| 能力 | 说明 |
|------|------|
| **跳跃机制** | 一段跳（地面）+ 二段跳（空中），无高度调节（固定跳跃高度） |
| **奔跑速度** | 自动奔跑，中等速度（适配休闲玩家反应时间） |
| **空中控制** | 有限水平微调（帮助玩家落地更精确） |
| **土狼时间** | 建议实现（走出平台边缘后约 0.1-0.15 秒内仍可跳跃，提升手感） |

**手感要求：**
- 跳跃响应时间 <100ms
- 二段跳在空中任意时刻可触发
- 落地缓冲期允许快速连续跳跃

---

### Combat System

**Square Fireline 为纯躲避型游戏，无传统战斗系统。**

| 元素 | 设计决策 |
|------|---------|
| **攻击机制** | 不适用 |
| **连击系统** | 不适用（MVP 阶段） |
| **敌人 AI** | 静态障碍物（石头、尖刺）+ 可能的动态障碍（后续阶段） |
| **击中反馈** | 死亡即重置，无血量扣除过程（MVP 简化设计） |

---

### Level Design Patterns

**无尽跑酷类型的平台跳跃设计：**

| 设计元素 | 说明 |
|---------|------|
| **平台跳跃挑战** | 空隙宽度、高度变化、连续障碍 |
| **地形起伏** | Perlin Noise 生成自然起伏，相邻 Chunk 高度差≤1 |
| **起始保护区** | 前 3 个 Chunk 为平坦地形，让玩家建立信心 |
| **障碍物密度** | 默认 30% 生成概率，空隙 10% 概率 |
| **检查点** | 每个 Chunk 可作为潜在重生点 |
| **难度曲线** | 随生存时间增加障碍物密度和地形复杂度 |

---

### Player Abilities and Unlocks

**能力系统（MVP 阶段）：**

| 类别 | 内容 |
|------|------|
| **起始能力** | 一段跳、二段跳、自动奔跑 |
| **可解锁能力** | MVP 阶段无解锁，所有能力初始可用 |
| **能力协同** | 二段跳 + 空隙跨越、连续跳跃节奏 |
| **升级路径** | MVP 阶段无升级系统 |

**未来阶段可选扩展（P1/P2）：**
- 连击系统（连续跳跃/躲避奖励）
- 收集品系统（金币、道具）
- 能力解锁（冲刺、滑翔等）

## Progression and Balance

### Player Progression

**进度系统：**

| 进度类型 | 说明 |
|---------|------|
| **分数进度** | 玩家通过生存时间获得分数，追求个人最高分是核心进度驱动力 |
| **技能进度** | 玩家通过反复练习掌握二段跳时机、连续跳跃节奏、障碍物预判 |
| **收集进度** | 金币和收集品系统（P2 阶段） |

**进度节奏：**
- **短期进度**：每次运行分数持续增长，玩家能清晰看到进步
- **中期进度**：打破个人最高分，获得成就感
- **长期进度**：收集品收集完成度（P2 阶段）

---

### Difficulty Curve

**难度曲线设计：**

| 阶段 | 说明 |
|------|------|
| **起始保护区** | 前 3 个 Chunk 为平坦地形，让玩家建立信心和熟悉操作 |
| **初期（0-30 秒）** | 稀疏障碍物，简单地形起伏，让玩家进入状态 |
| **中期（30 秒 -2 分钟）** | 障碍物密度增加，地形起伏加大，开始出现连续挑战 |
| **后期（2 分钟+）** | 高密度障碍物，复杂地形组合，考验玩家极限操作 |

**挑战缩放机制：**
- **障碍物密度**：默认 30% → 可逐步提升至 50%+
- **地形起伏**：高度差从 0 逐渐增加到最大值 1（系统限制）
- **空隙出现**：10% 概率 → 可逐步提升
- **连续障碍**：单个障碍 → 连续 2-3 个障碍组合

**难度选项：**
- MVP 阶段：单一难度曲线（面向休闲玩家优化）
- 未来可选：动态难度调整（根据玩家表现自动缩放）

**经济系统（P2 阶段）：**

| 资源 | 获取方式 | 用途 |
|------|---------|------|

**经济循环：**

```
运行游戏 → 收集金币 → 死亡/结束 → 金币保存 → 消费解锁外观 → 继续游戏
```

**MVP 阶段：** 无经济系统，专注核心玩法

## Level Design Framework

### Structure Type（结构类型）

| 元素 | 设计决策 |
|------|---------|
| **生成方式** | 程序化生成（Procedural Generation）- 每次运行游戏都会生成新的地图布局 |
| **关卡结构** | 无尽模式（Endless Runner）- 没有固定终点，持续生成直到玩家死亡 |
| **基础单元** | Chunk（20×5 瓦片）- 程序化生成的基本单位，动态生成/清理 |
| **地形算法** | Perlin Noise - 创造自然流畅的地形起伏，相邻 Chunk 高度差限制≤1 |
| **起始保护** | 前 3 个 Chunk 为平坦地形，让玩家建立信心 |

**技术优势：**
- 每次游戏体验都是独特的，提供无限重玩价值
- 动态 Chunk 系统优化内存占用
- Tile 翻转技术（单个 Tile 4 种翻转）创造视觉变化，优化资源使用

---

### Level Types（关卡类型/生物群系）

| 生物群系 | 视觉主题 | 解锁方式 | 特色元素 |
|---------|---------|---------|---------|
| **草地（Grassland）** | 绿色草地、森林背景 | 初始解锁 | 标准地形，适合新手 |
| **沙漠（Desert）** | 黄沙、仙人掌、金字塔背景 | 分数解锁 | 可能出现流沙区域、移动平台 |
| **雪地（Snowland）** | 冰雪、雪松、冰山背景 | 分数解锁 | 可能出现冰面（摩擦力降低）、雪球障碍 |

**生物群系切换机制：**
- 通过达成特定分数目标解锁新群系
- 玩家可在世界地图/选择界面选择已解锁的群系开始游戏
- 每个群系保持相同的玩法核心，主要变化为视觉主题和少量特殊元素

---

### Level Progression（关卡进度）

#### 游戏内流程（单局）
```
主界面 → 世界地图/选择界面 → 开始游戏 → 程序化生成地图 → 持续奔跑/跳跃 → 死亡 → 重生/返回主界面
```

#### 游戏流程设计

| 阶段 | 说明 | 设计目的 |
|------|------|---------|
| **主界面** | 简洁的 UI，只有"开始游戏"按钮 | 即点即玩，减少认知负担 |
| **世界地图/选择界面** | 展示已解锁的生物群系，选择起点环境 | 提供选择感，展示进度 |
| **开始游戏** | 无新手教程，直接开始 | 通过实践学习，30 秒内理解操作 |
| **游戏进行中** | 程序化生成无尽地图 | 每次体验独特，提供无限重玩价值 |
| **死亡后** | 立即重生到检查点 | 减少挫败感，保持节奏 |

#### 长期进度（跨局）

| 进度类型 | 解锁条件 | 奖励 |
|---------|---------|------|
| **沙漠群系** | 单局生存达到 60 秒 | 新视觉主题、新障碍元素 |
| **雪地群系** | 单局生存达到 120 秒 | 新视觉主题、冰面机制 |
| **更高难度** | 所有群系通关 | 可选动态难度调整 |

**进度设计原则：**
- 无传统关卡选择，通过生物群系解锁提供进度感
- 每次运行都是新的程序生成，无固定路线
- 追求个人最高分是核心驱动力

---

### Level Design Principles（关卡设计原则）

| 原则 | 说明 | 应用场景 |
|------|------|---------|
| **可读性优先** | 障碍物与背景对比度明显，玩家能清晰识别威胁 | 所有障碍物设计 |
| **公平性** | 所有死亡都应是玩家可预判、可避免的，无必死局面 | 地形/障碍生成逻辑 |
| **节奏感** | 紧张与放松交替，避免连续高压导致玩家疲劳 | 障碍物密度控制 |
| **学习曲线** | 新元素首次出现时难度最低，逐步增加复杂度 | 新障碍类型引入 |
| **动态难度** | 根据玩家表现微调生成概率，保持心流状态 | 后期可选扩展 |

**生成规则约束：**
- 相邻 Chunk 高度差≤1，保证可玩性
- 空隙宽度不超过玩家最大跳跃距离
- 障碍物间距保证至少 2 秒反应时间
- 起始保护区（前 3 Chunk）无障碍，让玩家熟悉操作

## Art and Audio Direction

### Art Style（艺术风格）

**风格定义：** 可爱像素风（Cute Pixel Art）- 现代技法打造的精致像素艺术，兼具复古情怀与当代审美。

| 元素 | 设计决策 |
|------|---------|
| **艺术风格** | 可爱像素风 - 简洁、亲和的像素美学，符合休闲游戏定位 |
| **参考作品** | 《Celeste》（精致像素动画）、《Stardew Valley》（亲和像素）、《Forager》（明亮简洁） |
| **色彩调色板** | 明亮、饱和度适中的色彩，友好非压抑；不同生物群系有明确色彩区分 |
| **视角** | 2D 横板视角，经典平台跳跃布局 |
| **角色设计** | 简洁可爱的几何造型（呼应"Square"名称），清晰轮廓便于识别 |

#### 生物群系视觉区分

| 群系 | 主色调 | 视觉特征 |
|------|--------|---------|
| **草地** | 绿色系 | 翠绿草地、深绿树木、温暖阳光 |
| **沙漠** | 黄色/橙色系 | 金黄沙地、橙红岩石、暖色调 |
| **雪地** | 白色/蓝色系 | 雪白地面、冰蓝装饰、冷色调 |

---

### Audio and Music（音频与音乐）

| 元素 | 设计决策 |
|------|---------|
| **音乐风格** | Chiptune/8-bit - 复古游戏感，活泼轻快 |
| **音乐 mood** | 轻松、积极、富有活力，符合休闲游戏节奏 |
| **音效设计** | 简洁有力的反馈音 - 跳跃、碰撞、UI 反馈清晰可辨 |
| **语音/对话** | 无语音设计 - 纯音效传达反馈，降低本地化成本 |

#### 音效需求清单

| 音效类型 | 说明 | 风格 |
|---------|------|------|
| **跳跃音效** | 地面起跳、二段跳（可选区分） | 简洁、轻快 |
| **碰撞音效** | 玩家触碰障碍物死亡 | 简短、不刺耳 |
| **UI 音效** | 按钮点击、分数增加 | 清脆反馈 |
| **得分音效** | 打破纪录提示 | 积极、奖励感 |
| **环境音效** | 风声、背景氛围（可选） | subtle、不干扰 |

---

### Aesthetic Goals（美学目标）

| 游戏支柱 | 艺术/音频支持方式 |
|---------|-----------------|
| **简单易上手** | 清晰视觉层次，障碍物与背景高对比度；音效反馈即时可辨 |
| **流畅操作** | 像素动画流畅（60FPS 适配）；跳跃音效响应<100ms |
| **无限重玩价值** | 程序化地图 + 视觉变化 Tile；音乐循环不枯燥 |
| **快速重试** | 死亡视觉/音频反馈简洁不挫败；重生即时 |

### 平台适配考虑

| 平台 | 艺术/音频适配 |
|------|------------|
| **PC** | 支持高分辨率像素缩放（整数倍） |
| **移动端** | 像素密度适配，色彩在移动屏幕上保持鲜明 |
| **主机** | 4K 输出适配，像素完美缩放 |

## Technical Specifications

### Performance Requirements（性能需求）

| 目标 | 数值 | 测量方式 |
|------|------|---------|
| **帧率目标** | 60FPS 稳定 | 性能分析工具监测，全场景覆盖 |

| **加载时间** | 冷启动 <3 秒 | 从点击图标到主界面可交互时间 |

| **内存占用** | 控制在合理范围，无内存泄漏 | 内存分析工具监测 |

- **帧率优先于画质** - 作为平台跳跃游戏，流畅操作体验至关重要

- **移动端优化** - 热管理和电池寿命考虑

## Development Epics

### Epic Overview

| # | Epic 名称 | 范围 | 依赖 | 估算故事数 |
|---|---------|------|------|-----------|
| 1 | **核心玩家控制器** | 玩家移动、一段跳、二段跳、物理碰撞 | 无 | 5-8 |
| 2 | **基础地图系统** | Tilemap、Chunk 动态生成、基础地形 | 无 | 6-10 |
| 3 | **障碍物与碰撞** | 障碍物生成、碰撞检测、死亡逻辑 | Epic 1, 2 | 5-8 |
| 4 | **游戏流程管理** | 游戏状态、分数系统、检查点重生 | Epic 3 | 4-6 |
| 5 | **UI 系统** | 主界面、分数显示、最高分记录 | Epic 4 | 4-6 |
| 6 | **地图生成扩展** | 生物群系、难度曲线、群系解锁 | Epic 2, 4 | 6-10 |
| 7 | **音频系统** | BGM、音效（跳跃/碰撞/UI） | Epic 4 | 3-5 |
| 8 | **Polish 与优化** | 粒子效果、动画优化、性能调优 | 所有以上 | 5-8 |

---

### Recommended Sequence（推荐执行顺序）

```
Epic 1 (核心玩家控制器) → Epic 2 (基础地图系统) → Epic 3 (障碍物与碰撞)
                                                            ↓
Epic 8 (Polish 与优化) ← Epic 7 (音频系统) ← Epic 6 (地图扩展) ← Epic 5 (UI 系统) ← Epic 4 (游戏流程)
```

**执行顺序理由：**

1. **Epic 1-2 并行启动** - 玩家控制和基础地图是独立的基础系统，可并行开发
2. **Epic 3 需要前两者** - 障碍物需要玩家和地图都存在才能测试
3. **Epic 4 完成垂直切片** - 游戏流程 + 分数 + 重生 = 第一个可玩版本
4. **Epic 5-7 增强体验** - UI、音频、扩展内容让游戏更完整
5. **Epic 8 最后打磨** - 在所有功能稳定后进行 polish

---

### Vertical Slice（垂直切片）

**第一个可玩里程碑：Epic 4 完成后**

**包含内容：**
- 完整的玩家控制（移动、一段跳、二段跳）
- 基础地形生成（单一生物群系，无障碍）
- 基础障碍物（石头、尖刺）
- 碰撞检测与死亡逻辑
- 分数系统（基于生存时间）
- 立即重生功能

**可玩体验：**
```
点击开始 → 角色自动奔跑 → 玩家跳跃躲避障碍 → 死亡 → 立即重生 → 继续游戏 → 追求高分
```

**不包含内容（后续 Epic）：**
- 主界面美化（Epic 5）
- 生物群系解锁（Epic 6）
- 背景音乐/音效（Epic 7）
- 粒子效果、动画 polish（Epic 8）

**验收标准：**
- 玩家可以流畅完成一次完整的游戏循环
- 二段跳手感调优完成
- 输入延迟 <100ms
- 稳定 60FPS
- 无崩溃、无阻塞性 Bug

## Assumptions and Dependencies

### Key Assumptions

#### 技术假设
| 假设 | 风险等级 | 缓解方案 |
|------|---------|---------|
| Unity 2022.3 LTS 将保持稳定 | 低 | 使用 LTS 版本，避免测试版 |
| AI 辅助开发工具持续可用 | 中 | 保持对生成代码的深度理解，手动优化能力 |
| Tilemap 系统性能满足无尽生成需求 | 低 | 动态 Chunk 清理机制保证内存 |

#### 团队假设
| 假设 | 风险等级 | 说明 |
|------|---------|------|
| 单人开发 + AI 辅助 | 中 | 充分利用 AI 辅助开发提高效率 |
| 无外部承包商依赖 | 低 | 可能使用 Asset Store 现成资产加速开发 |
| BMad 方法工作流可用 | 低 | 结构化开发框架保证进度 |

| 假设 | 风险等级 | 说明 |
