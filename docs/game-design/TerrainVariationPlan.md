# 地形起伏生成方案

> 版本：1.0
> 创建日期：2026-03-13
> 状态：待实施

---

## 一、设计目标

为程序化生成的横板跳跃地图添加**自然的地形起伏**，提升游戏可玩性和视觉层次感。

---

## 二、核心概念

### 2.1 什么是"地形起伏"

在现有的 Tilemap 网格系统中，**每列地块的高度可以不同**，形成阶梯状的地形变化。

```
原设计（平坦）：
┌──┬──┬──┬──┬──┬──┐
│草│草│草│草│草│草│  ← 所有草坪在同一高度
├──┼──┼──┼──┼──┼──┤
│土│土│土│土│土│土│
└──┴──┴──┴──┴──┴──┘

新设计（起伏）：
┌──┬──┬──┬──┬──┬──┐
│  │  │  │  │草│  │
│  │  │  │草├──┤  │
│  │  │草├──┤土│  │
│  │草├──┤土│土│  │
│草├──┤土│土│土│  │
├──┼──┼──┼──┼──┼──┤
│土│土│土│土│土│土│
└──┴──┴──┴──┴──┴──┘
```

### 2.2 设计原则

| 原则 | 说明 |
|------|------|
| **可通过性优先** | 相邻列高度差 ≤ 1，确保玩家可通过 |
| **自然过渡** | 使用噪声算法，避免生硬的阶梯 |
| **可控随机** | 相同种子生成相同地形，便于调试 |
| **向后兼容** | 可开关，不影响现有平坦地图 |

## 三、地形参数定义

### 3.1 基础参数

| 参数名 | 含义 | 建议值 | 说明 |
|--------|------|--------|------|
| **基准高度** | 地形起伏的中心线高度 | 5 | 所有高度计算的基准 |
| **最小高度** | 地形高度下限 | 2 | 防止地形过低 |
| **最大高度** | 地形高度上限 | 8 | 防止地形过高 |
| **起伏幅度** | 高度变化的最大范围 | 3 | 值越大，起伏越剧烈 |

### 3.2 噪声参数

| 参数名 | 含义 | 建议值 | 说明 |
|--------|------|--------|------|
| **噪声频率** | 控制起伏的密集程度 | 0.1~0.2 | 值越小，起伏越平缓 |
| **噪声强度** | 控制高度变化的幅度 | 1.0 | 0=无变化，1= full 变化 |
| **随机种子** | 地形生成的种子值 | 任意整数 | 相同种子=相同地形 |

### 3.3 保护参数

| 参数名 | 含义 | 建议值 | 说明 |
|--------|------|--------|------|
| **平坦 Chunk 数** | 起始区域的平坦 Chunk 数量 | 3 | 给玩家适应时间 |

## 四、地形生成规则

### 4.1 高度计算流程

```
Step 1: 读取 X 坐标
         ↓
Step 2: 计算噪声值 (Perlin Noise)
         ↓
Step 3: 映射到目标高度范围
         ↓
Step 4: 检查与前一列的高度差
         ↓
Step 5: 如高度差 > 1，进行平滑修正
         ↓
Step 6: 限制在 [最小高度，最大高度] 范围内
         ↓
Step 7: 输出最终高度
```

### 4.2 高度差限制规则

```
规则：相邻列高度差 ≤ 1

示例：
x=0, 高度=5  ✓
x=1, 高度=6  ✓ (差=1)
x=2, 高度=8  ✗ (差=2，修正为 7)
x=3, 高度=7  ✓ (差=1)

修正后：
x=0, 高度=5
x=1, 高度=6
x=2, 高度=7
x=3, 高度=7
```

### 4.3 起始保护规则

```
前 N 个 Chunk（平坦区域）：
- 所有列高度 = 基准高度
- 不生成起伏
- 给玩家适应时间

第 N+1 个 Chunk 开始：
- 逐步引入高度变化
- 过渡 Chunk 的高度差额外限制
```

## 五、视觉表现

### 5.1 地块层级结构

```
单列结构（从下到上）：

  ┌──────┐
  │ 草坪  │  ← 顶层（始终 1 格）
  ├──────┤
  │ 土壤  │  ← 中间层（高度 -1 格）
  │  :   │
  │  :   │
  ├──────┤
  │ 土壤  │  ← 底层（y=0）
  └──────┘
```

### 5.2 地块对齐规则

| 规则 | 说明 |
|------|------|
| **垂直对齐** | 每列地块严格对齐网格，无偏移 |
| **水平对齐** | 相邻地块边缘无缝衔接 |
| **边界对齐** | 草坪层与土壤层右边界对齐 |

### 5.3 Tile 边界规则

在地形起伏时，Tile 的侧面可能暴露（挨着空气）或被遮挡（挨着土壤）。暴露的侧面需要有黑边，被遮挡的侧面应当无黑边以避免接缝。

#### 核心结论

| 层 | 是否需要黑边 Tile | 说明 |
|------|------|------|
| **草坪层** (y = columnHeight - 1) | **是** | 可能挨着空气 |
| **土壤层** (y = 0 ~ columnHeight - 2) | **否** | 高度差≤1 时，侧面一定是草坪或土壤 |

#### 草坪层 Tile 选择

| Tile | 左侧 | 右侧 | 使用条件 |
|------|------|------|----------|
| grass_middle | 无 | 无 | leftHeight ≥ columnHeight 且 rightHeight ≥ columnHeight |
| grass_left | 黑边 | 无 | leftHeight < columnHeight（左侧暴露） |
| grass_right | 无 | 黑边 | rightHeight < columnHeight（右侧暴露） |

**判定逻辑**：
```
1. 获取左侧列高度 (leftHeight) 和右侧列高度 (rightHeight)
2. 如果 leftHeight < 当前高度 → 左侧暴露 → 使用 grass_left
3. 如果 rightHeight < 当前高度 → 右侧暴露 → 使用 grass_right
4. 两侧都不暴露 → 使用 grass_middle
```

#### 土壤层 Tile 选择

土壤层**统一使用 dirt_full**（无黑边），因为在高度差≤1 的规则下，土壤层侧面永远不会挨着空气。

**证明**：
```
- 当前列高度 = H，土壤层 y 范围 = [0, H-2]
- 相邻列高度 = H-1 或 H 或 H+1（高度差≤1）
- 相邻列的草坪层 y = 相邻列高度 - 1 ≥ H-2

所以对于当前列土壤层的任意 y：
- 相邻列在相同 y 位置一定有草坪或土壤
- 土壤层永远不会直接挨着空气
```

#### 视觉效果示例

```
示例 1：斜坡向上（从左到右升高）

左侧高度=3，右侧高度=2

    ┌──────────┐
    │ grass_R  │  ← 右侧暴露（右边低），用 grass_right
    ├──────────┤
┌───┤ dirt_F   │  ← y=1，右侧挨着草坪，用 dirt_full
│草 │          ├──────┐
│中 │ dirt_F   │ 草坪 │  ← y=0，右侧挨着土壤，用 dirt_full
└───┴──────────┴──────┘

图例：
grass_R = grass_right（右侧黑边）
grass_M = grass_middle（无黑边）
dirt_F  = dirt_full（无黑边，现有 dirt_tile 可复用）


示例 2：斜坡向下（从左到右降低）

左侧高度=2，右侧高度=3

    ┌──────────┐
    │ grass_R  │  ← 右侧同高，右侧不暴露
    ├──────────┤
    │ dirt_F   │  ← y=1，右侧挨着土壤
    ├──────────┤
┌───┼──────────┼──────┐
│草 │  dirt_F  │ 土壤 │  ← y=0，两侧无暴露
│中 │          │      │
└───┴──────────┴──────┘


示例 3：山峰（中间高，两侧低）

        ┌──────────┐
        │ grass_LR │  ← 两侧暴露（可用 grass_left 或 grass_right 代替）
        ├──────────┤
    ┌───┤ dirt_F   ├───┐
    │草 │ dirt_F   │草 │
    │中 │          │中 │
    └───┴──────────┴───┘
```

#### Tile 资源需求

| Tile | 用途 | 优先级 | 来源 |
|------|------|--------|------|
| grass_middle | 无黑边草坪 | P0 | 现有 grass_middle |
| grass_left | 左侧黑边草坪 | P0 | 现有 grass_left（重新定义用途） |
| grass_right | 右侧黑边草坪 | P0 | 现有 grass_right（重新定义用途） |
| dirt_full | 无黑边土壤 | P0 | 现有 dirt_tile（可复用） |

**备注**：
- `grass_left/grass_right` 重新定义为"暴露边界"而非"Chunk 边界"
- 土壤层不需要新增 Tile，现有 `dirt_tile` 可直接复用
- `dirt_edge_LR` 不需要（土壤层不会暴露）

## 六、障碍物生成规则

### 6.1 障碍物位置

```
障碍物生成在草坪层的正上方（y = 列高度）

示例：
        ┌──┐
        │障│  ← 障碍物
    ┌──┼──┼──┐
    │草│草│草│  ← 草坪层
    ├──┼──┼──┤
    │土│土│土│
    └──┴──┴──┘
```

### 6.2 障碍物生成限制

| 条件 | 规则 |
|------|------|
| **高度变化区域** | 高度差 > 0 的列不生成障碍物 |
| **间隔限制** | 与前一个障碍物的距离 ≥ 最小间隔 |
| **概率限制** | 按配置的生成概率判定 |

## 七、配置方案

### 7.1 配置资产结构

```
ScriptableObject: TilemapMapConfig

字段分组：
├── Chunk 尺寸
│   ├── chunkWidth: 20
│   └── groundHeight: 5（改为 baseHeight）
├── 生成配置
│   ├── aheadChunkCount: 3
│   └── behindChunkCount: 2
├── 地形起伏（新增分组）
│   ├── enableTerrainVariation: false
│   ├── baseHeight: 5
│   ├── minHeight: 2
│   ├── maxHeight: 8
│   ├── heightVariation: 3
│   ├── noiseFrequency: 0.15
│   ├── noiseStrength: 1.0
│   ├── seed: 12345
│   └── flatChunkCount: 3
├── 草坪 Tile
│   ├── grassLeft
│   ├── grassMiddle
│   └── grassRight
├── 土壤 Tile
│   └── dirtTile
└── 障碍物配置
    ├── obstacleTiles[]
    ├── obstacleSpawnChance
    ├── minObstacleGap
    └── maxObstacleGap
```

### 7.2 推荐配置预设

#### 预设 A：平缓地形（适合休闲模式）
```
baseHeight: 5
minHeight: 4
maxHeight: 6
heightVariation: 2
noiseFrequency: 0.08
flatChunkCount: 5
```

#### 预设 B：标准地形（默认推荐）
```
baseHeight: 5
minHeight: 3
maxHeight: 7
heightVariation: 3
noiseFrequency: 0.15
flatChunkCount: 3
```

#### 预设 C：挑战地形（适合困难模式）
```
baseHeight: 5
minHeight: 2
maxHeight: 8
heightVariation: 4
noiseFrequency: 0.2
flatChunkCount: 2
```

## 九、风险与注意事项

### 9.1 技术风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 高度缓存无限增长 | 内存泄漏 | 清理 Chunk 时同步清理缓存 |
| 障碍物生成在空中 | 游戏 bug | 高度变化区域禁止生成障碍物 |
| 地形无法通过 | 卡关 | 严格限制高度差 ≤ 1 |

### 9.2 设计风险

| 风险 | 影响 | 缓解措施 |
|------|------|----------|
| 起伏过于频繁 | 玩家疲劳 | 调整 noiseFrequency |
| 起伏过于剧烈 | 无法跳跃 | 限制 maxHeight - minHeight |
| 起始区域太短 | 新手不适应 | 增加 flatChunkCount |

## 十、验收标准

### 10.1 功能验收

- [ ] 开关可控制起伏启用/关闭
- [ ] 地形高度在 [minHeight, maxHeight] 范围内
- [ ] 相邻列高度差 ≤ 1
- [ ] 起始 N 个 Chunk 保持平坦
- [ ] 障碍物生成在草坪层上方
- [ ] 清理 Chunk 时同步清理缓存

### 10.2 视觉验收

- [ ] 地块严格对齐网格
- [ ] 草坪与土壤边界对齐
- [ ] 无缺口/悬空地块
- [ ] 地形过渡自然
- [ ] Tile 边界无黑边接缝

### 10.3 性能验收

- [ ] 无明显帧率下降
- [ ] 内存占用稳定
- [ ] 无内存泄漏
